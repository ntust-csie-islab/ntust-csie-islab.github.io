  1. Abstract:
    1. Relevance feedback mechanisms → 在 search result 中，user 將 有 興 趣 的 page 做 記 號 ，system 從 這 些 page 中 翠 取 一 些 keyword ， feedback 給系統.
    2. Hypertext mapping technique → the links among the hit pages Vector space retrieval model
    3. Query sharing → allow any user to save his query statement on the server's side(reuse it)
    4. Vector space retrieval model

  2. System Architecture:

    3. Indexer robot(autonomous WWW browser)
    → 建 立 及 維 護 index database for keyword searching 工作：visits a given WWW site
    retrieves WWW pages
    extract keywords and hyperlink data
    insert keywords into an index
    Index Database(Hashing table)
     Table：Page-ID table : URL → pageID
    Keyword-ID table : keyword → keywordID
    Page modification-date table : pageID → date (用於 periodic maintenance)
    Hyperlink Table : pageID → Incoming hyperlink，Outcoming hyperlink
    Inverted index table : keywordID → <pageID,word position>
    (used in query) Forward index table : pageID→keywordID
    (利用一組文件表示一個keyword)
    Periodic maintenance
     Index robot checks the validity of every URL entry
    Whether the page has been modified
     Search engine
    利用TFxIDF search→the relevance score of document(I) with respect to query Q
    ……Eequation1
    ( It's the sum of the weights of the query terms appearing in the document )
     ……Eequation2
    TF(i,j) : the term frequency of term(j) in document(i)
    TF(i,max) : the maximum term frequency of a keyword in document(i)
    N : the number of documents in the collection
    DF(j) : the number of documents containing term(j)
     Each component of such a vector corresponds to the weight of term in query or document
    Point：
     Equation1 : the term weight is normalized for document length (short documents have equal chances to be retrieved as longer documents)
    Equation2 : 給 予 在 少 部 分 文 件 中 常 出 現 的 keyword 較 高 的 weight
